# 零售业务SQL数据分析完整作品集
## 项目总述
### 数据模型
采用标准星型模型：
- 事实表：销售订单表（每一笔交易流水，存储度量值：销量、单价）
- 维度表：门店、商品、客户（存储描述性属性，用于分组、筛选）
### 开发规范
1. 全部SQL采用通用标准语法，兼容SQLite/MySQL；
2. 禁用不兼容写法，如GROUP BY 表.*；
3. 每个分析模块独立Jupyter Notebook，无跨文件依赖；
4. 每个Notebook统一结构：业务需求 → 分析思路 → 核心知识点 → 可执行代码 → 结果业务解读。

## 模块1：01_overview_agg.ipynb 全量业务核心指标一键统计
### 业务需求
基于四张原始业务表，一次性输出项目整体业务规模、数据时间范围等核心大盘指标，作为数据分析前置探查步骤。
### 思路分析
1. 分别对四张表做基础聚合，使用UNION ALL整合多行指标结果；
2. 使用COUNT、COUNT(DISTINCT)区分总量与唯一量；
3. MIN/MAX获取订单数据起止时间，确认数据周期。
### 核心知识点
COUNT/COUNT(DISTINCT)/MIN/MAX基础聚合、UNION ALL结果合并、业务数据探查方法论
### 业务价值
快速掌握项目整体体量，为后续细分维度分析划定数据边界。

## 模块2：02_prov_category_group.ipynb 省份-品类交叉营收分布统计
### 业务需求
关联订单、门店、商品表，统计每个省份下各商品品类的营收总额与营收占比，分析区域消费偏好。
### 思路分析
1. 三表INNER JOIN关联事实与维度表；
2. 按省份、商品类别双维度分组聚合；
3. 子查询计算全局总营收，用于各品类占比换算，保留两位小数格式化。
### 核心知识点
多表关联、多维度GROUP BY、聚合计算百分比、子查询作为全局统计分母
### 业务价值
指导区域供应链备货、分区域营销活动投放。

## 模块3：03_month_trend_lag.ipynb 月度经营趋势与环比增长分析
### 业务需求
按月汇总订单、营收、客单价，计算每月营收环比，判断业务增长/下滑趋势，过滤小额异常订单。
### 思路分析
1. STRFTIME提取年月实现时间分组；
2. CTE封装月度基础指标；
3. LAG()窗口函数获取上一期营收，计算环比增长率；
4. WHERE过滤单笔10元以下无效小额订单。
### 核心知识点
SQLite日期函数、LAG偏移窗口函数、环比计算、WHERE过滤异常数据
### 业务价值
月度经营复盘，定位业绩波动节点。

## 模块4：04_shop_rank_three_window.ipynb 分省份门店好评率多逻辑排名
### 业务需求
筛选有交易记录的门店，在各省份内部按照好评率排序，同时展示ROW_NUMBER、RANK、DENSE_RANK三种排名结果，区分三者使用场景。
### 思路分析
1. INNER JOIN过滤无订单门店，DISTINCT去重门店数据；
2. CTE封装门店基础信息；
3. PARTITION BY按省份分区，每个省份独立排名；
4. 三个排名函数并列输出直观对比差异。
### 核心知识点
INNER JOIN数据过滤、DISTINCT去重、CTE、PARTITION BY分区、三大排名窗口函数
### 业务价值
区域门店评优、筛选标杆门店复制运营经验。

## 模块5：05_customer_tile_ntile.ipynb 客户价值分层与消费画像
### 业务需求
依据客户累计消费总额，将全体客户分为高/中/低价值三层，统计每层客户数量、总消费、平均订单频次。
### 思路分析
1. 订单关联客户表，按客户ID聚合总消费；
2. NTILE(3)窗口函数均分三层消费人群；
3. CASE WHEN给分层打业务标签，再聚合分层指标。
### 核心知识点
NTILE分桶窗口函数、CASE条件标签、多层CTE嵌套、用户分层模型
### 业务价值
差异化会员运营：高价值客户维护、低价值客户唤醒。

## 模块6：06_customer_exists_high_spend.ipynb EXISTS存在匹配——大额消费客户全订单明细查询
### 业务需求
筛选名下存在单笔订单营收≥5000元的客户，输出该客户所有历史订单、门店、商品、会员信息，用于高消费潜力客户分层运营。
### 思路分析
1. 使用EXISTS做存在性匹配：只要客户任意一笔订单金额达到5000，该客户全部订单都会被筛选出来；
2. 仅做行级条件判断，无多层聚合嵌套查询，执行逻辑简单清晰；
3. 采用星型模型四表关联，带出门店、商品、会员等业务可读维度；
4. 采用复合排序，优先按客户ID分组，同客户内订单按日期倒序，方便完整查看单人消费轨迹。
### 核心知识点
EXISTS半连接存在性查询、短路扫描机制、星型模型多表INNER JOIN关联、数值条件过滤、多字段复合排序逻辑
### 业务价值
定位高潜力消费客群，完整分析其全周期消费习惯，针对性推出高端商品专属营销活动。

## 模块8：08_shop_moving_avg_window.ipynb 门店月度销售额累计与移动平均
### 业务需求
统计每个门店每月销售额，计算累计销售额、近3个月移动平均销售额，分析门店长期经营走势。
### 思路分析
1. 门店+月份双维度聚合月度业绩；
2. SUM() OVER 分区累计求和；
3. AVG() OVER 指定前后两行窗口计算三月移动均值。
### 核心知识点
分区累计窗口、滑动窗口、时间序列高级分析
### 业务价值
门店长期业绩评估、销售预测参考。
